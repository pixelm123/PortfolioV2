Signature Project

Expense Tracker API

A production REST API I built as a reference for how I think a backend should be structured. Not just a CRUD app - every architectural decision was deliberate and every layer has a clear reason to exist.

Clean Architecture - domain logic is completely isolated from infrastructure. You can swap the database without touching a single business rule.
CQRS with MediatR - reads and writes are separated so each operation is independently testable and changes don't bleed into each other.
Background service that monitors budgets and fires alert emails the moment a threshold is crossed.
66 unit tests using xUnit and Moq - every command, every query, edge cases and error paths, not just the happy path.
RFC 7807-compliant error responses throughout, so clients always know exactly what went wrong and why.
